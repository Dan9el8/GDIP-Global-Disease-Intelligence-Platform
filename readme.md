## Key Features

| Feature | Description |
|---------|-------------|
|  **Global Data Aggregation** | Ingests outbreak reports from WHO, CDC, Ministries of Health, and open datasets every 6 hours. |
|  **Predictive Analytics** | XGBoost, Random Forest, and LSTM models forecast cases 4 weeks ahead with confidence intervals. |
|  **Smart Alerts** | Email, SMS, dashboard, or webhook when cases spike, mortality rises, or risk score exceeds thresholds. |
|  **Interactive Heatmap** | Real‑time global map with cluster markers, risk colouring, and historical overlays. |
|  **AI Report Generator** | Weekly human‑readable reports (Markdown / PDF) generated by GPT‑4o or open‑source LLMs. |
|  **RAG Assistant** | Ask natural language questions – the assistant answers using WHO/CDC publications and live data. |
|  **Africa‑First Forecasting** | Pre‑tuned models for malaria, cholera, dengue, and mpox using African historical data (2015–2025). |
|  **Satellite + Weather** | MODIS vegetation indices and ERA5 climate data improve risk prediction. |
|  **Real‑time Streams** | Apache Kafka powers low‑latency outbreak event processing. |
|  **Multi‑tenant Workspaces** | Isolated environments for governments, NGOs, and research institutions. |
|  **Public REST API** | Rate‑limited endpoints for third‑party developers (no API key required for read‑only data). |

##  Quick Start (Local Development)

### Prerequisites
- Node.js 20+
- Python 3.11+
- Docker & Docker Compose
- MongoDB 7+ (or use the Docker container)

### Clone & Install
```bash
git clone https://github.com/dan9el8/GDIP-Global-Disease-Intelligence-Platform.git
cd gdip

# Backend
cd backend
npm install
cp .env.example .env   # edit with your MongoDB URI, API keys
npm run dev

# Frontend (second terminal)
cd ../frontend
npm install
npm run dev

# ML Service (third terminal)
cd ../ml-service
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app:app --reload --port 8001
```

Open [http://localhost:5173](http://localhost:5173) – you’re ready!

### Using Docker (Easier)
```bash
docker-compose up -d
```
Services will be available at:
- Frontend: `http://localhost:80`
- API: `http://localhost:3000`
- ML Service: `http://localhost:8001`

##  How to Contribute

We welcome contributors of all skill levels! Please read our [CONTRIBUTING.md](CONTRIBUTING.md) first.

### Good First Issues
Look for labels `good first issue` and `help wanted` in the [issue tracker](https://github.com/dan9el8/GDIP-Global-Disease-Intelligence-Platform/issues).

### Contribution Workflow
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'Add some amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request against `main`.

### Development Guidelines
- **Code style**: ESLint + Prettier (frontend), Black + isort (Python).
- **Tests**: Jest for frontend, Mocha/Chai for backend, pytest for ML service.
- **Commit messages**: Follow [Conventional Commits](https://www.conventionalcommits.org/).

All contributors must abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
